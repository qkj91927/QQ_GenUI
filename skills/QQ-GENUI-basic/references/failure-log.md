# QQ-GENUI Basic Failure Log

> 目的：记录 `QQ-GENUI-basic` 在真实任务中的失败模式、触发原因、修复动作与预防规则。
>
> 使用方式：每次执行本 skill 前先通读最近记录，并检索与当前任务最相似的条目；若本次出现失败、偏差或需要用户反复调整，任务结束后必须追加新条目。

## 记录范围

满足以下任一情况时，必须回填日志：

- agent 首次生成失败，无法直接满足用户目标
- agent 虽完成初稿，但用户需要重复指出问题并多轮调整
- 输出违反 QQ_GenUI 规范（组件细则、结构语义、版式细节、图标本地化、交互行为）
- 漏读 SPEC、误读 SPEC 或未完整实现 SPEC 中描述的交互
- 图标、本地资源、背景色、状态栏、组合关系等反复出现相同错误

## 追加规则

- 每次新增记录追加到文件顶部，确保最近问题最先被读取
- 记录必须具体到"错误现象 → 根因 → 修复动作 → 下次预防规则"
- 不写空泛总结，如"细心一点"、"下次注意"
- 若同类错误再次发生，在旧条目下补充"复发说明"，或新增一条更具体的记录

## 条目模板

```md
## [日期] 标题
- **任务类型**: 页面生成 / 局部调整 / 合规修复 / 其它
- **触发场景**: 用户原始需求摘要
- **失败表现**: 首次输出哪里失败，或用户为什么需要反复调整
- **根因分析**: 漏读了什么、误判了什么、哪个 Gate 执行不到位
- **解决方式**: 最终如何修复
- **预防规则**: 下次执行前必须新增的检查动作
- **关联组件/规范**: 涉及的组件名、SPEC 文件、Token、图标或布局规则
- **复发说明**: 无 / 第 N 次发生，和历史哪条类似
```
